# AINFINITY 第一届『饮料王』争霸赛 🥤

一个纯静态网页小游戏：**按一下开始计时，再按一下停在正好 10.000 秒**，越准越强。
决出最准的前两名当『饮料王』，由他们决定公司接下来半个月采购什么饮料。

- 纯静态单页（一个 `index.html`，内联 CSS/JS），无框架、无构建、无 npm。
- 排行榜用 [Supabase](https://supabase.com/)（CDN 引入，匿名 key 即可）。
- 两种模式：**自由练习**（不记名、不上榜）、**挑战赛**（房间 + 排行榜）。

## 一、配置 Supabase（约 5 分钟）

1. 注册并新建一个项目：<https://supabase.com/dashboard>。
2. 打开项目 → 左侧 **SQL Editor** → New query → 把 `schema.sql` **整段粘贴**进去 → **Run**。
   - 会建好 `rooms`、`scores` 两张表、唯一约束，并开启 RLS（匿名可读/插/改，不可删）。
3. 打开 **Project Settings → API**，拿两样东西：
   - **Project URL**（形如 `https://abcd1234.supabase.co`）
   - **anon public** key（很长的一串，**不是** service_role）
4. 编辑 `index.html` 顶部的 `CONFIG`：

   ```js
   SUPABASE_URL:      'https://abcd1234.supabase.co',
   SUPABASE_ANON_KEY: 'eyJhbGciOi...你的 anon key...',
   ```

> ⚠️ 只填 **anon** key。它是公开可见的（前端本来就会暴露），靠 RLS 限制权限，没问题。
> 千万别填 service_role key。

填好后挑战赛就能用了。**不填也行**——自由练习不依赖 Supabase，页面会提示挑战赛未配置。

### 其他可调项（都在 `CONFIG` 里）

- `TARGET_SECONDS`(10)、`DECIMALS`(3，可改 4 但第 4 位只是装饰)、`ATTEMPTS`(3)、`WINNER_COUNT`(2)
- `BRAND_TEAL`('#35B8B0')、`INK`('#1A1A1A') —— 全站颜色从这走
- `TIERS` —— 四档评价阈值（先放着，看到真实成绩分布后再调松紧）
- `VERDICTS` —— 四档评价文案，每档多写几条会随机抽

## 三、部署成链接（发给同事）

### Netlify（最省事）
- 把这个文件夹直接拖到 <https://app.netlify.com/drop>，秒出一个 `https://xxx.netlify.app` 链接。

1. 新建仓库，把这几个文件推上去（`index.html` 在根目录）。
2. 仓库 **Settings → Pages** → Source 选 `main` 分支、根目录 → Save。
3. 等一会儿拿到 `https://你的用户名.github.io/仓库名/` 链接。

部署后把链接发群里即可。

## 四、怎么玩 / 怎么组织一场比赛

### 自由练习
首页点「自由练习」→ 空格/点大按钮开始，再按一下停止 → 看成绩和评价 → 「再来一次」。
可选用 localStorage 记一个**本机最佳**自娱自乐（不上榜，可一键清除）。

### 挑战赛
1. **发起挑战（房主）**：首页 →「挑战赛」→ 填名字 →「生成房间」。
   - 进房后会看到两条链接：
     - **加入链接**（`.../?room=房间号`）—— 发到群里给同事。
     - **房主专属链接**（`.../?room=房间号&host=令牌`）—— **只发给自己**，只有它能结算/开新局。
   - 复制按钮一键复制。
2. **加入挑战**：同事点加入链接会自动带入房间号，填名字即可进；也可手动输房间号 + 名字。
3. **玩**：每人 **3 次**机会，自动取离 10 秒最近的那次记入排行榜。界面显示「机会 1/3」和当前最好成绩。
4. **排行榜**：按离 10 秒的绝对差升序，显示名次/名字/最好成绩（真实秒数）/差值，前两名高亮标 **👑 饮料王**。每 5 秒自动刷新，也有手动「刷新」。
5. **结算**：房主点「结算 / 公布结果」冻结排行榜，前两名即『饮料王』。
6. **开新一局**：房主点「开新一局（清空）」清空本房间成绩、重置每人 3 次机会，重新开打。

> 房主权限用 **host token** 轻量校验：只有带 token 的房主链接才会显示「结算 / 开新局」按钮。够内部娱乐用。

## 五、关于反作弊（说在前面）

这是低风险的内部娱乐，反作弊**够用就行**：

- 每个（房间号 + 名字）只占一条成绩，3 次用完后锁定（前端 localStorage 记次数 + 后端 `attempts` 计数）。
- **计时数字全程露在外面，理论上用脚本卡 10 秒、或换名字刷次数仍可能作弊。** 这个我们接受。

> 🏁 **真要较真**：让房主结算后，叫排行榜**前几名现场 / 投屏决赛**再按一次定胜负，当众玩谁也作不了弊。

## 六、自测清单

- [ ] 开两个浏览器（或一个正常窗口 + 一个无痕窗口），同一房间用两个不同名字各玩。
- [ ] 每人玩 3 次以上，确认**只取最好那次**、第 4 次起被锁定。
- [ ] 排行榜按**差值升序**排列，前两名有 **👑 饮料王** 高亮。
- [ ] 把加入链接复制到另一个浏览器打开，确认**自动带入房间号**。
- [ ] 用手机打开链接，确认**点底部大按钮能玩**。
- [ ] 玩的时候按空格，确认**页面不滚动**；长按空格不会连发。
- [ ] 房主点「结算」后榜单冻结、提交被挡；点「开新一局」后榜单清空、机会重置。

## 技术备注

- 计时用 `performance.now()` + `requestAnimationFrame` 平滑刷新；停止时取整毫秒计分。
- 计时数字用 `tabular-nums` 等宽数字，避免宽度跳动；颜色 = `CONFIG.BRAND_TEAL`。
- 状态机：`idle → running → stopped`；空格 `event.repeat` 过滤、`preventDefault` 屏蔽滚动。
- “开新一局”因 RLS 不开放 DELETE，改为把成绩 `attempts` 归零（前端只展示 `attempts > 0`），等效清空。
- 无障碍：按钮可聚焦、焦点可见、`prefers-reduced-motion` 下关闭动画。
